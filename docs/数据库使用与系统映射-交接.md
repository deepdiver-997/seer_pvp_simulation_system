本文档用于让后续 Agent 快速接手：

## 1. 一键更新与离线重建

数据库默认路径：`scripts/data/processed/seer_official.sqlite`

常用命令：

```bash
# 在线：抓取 + 导入
python3 scripts/update_seer_official_data.py

# 强制重下后导入
python3 scripts/update_seer_official_data.py --force

# 离线：只用本地 raw 重建 SQLite
python3 scripts/update_seer_official_data.py --skip-fetch
```

## 2. 代码里谁在用数据库

### 2.1 入口对象

- `official_data::OfficialDataStore`：单例持有只读仓库
- `official_data::OfficialDataRepository`：封装固定查询接口

关键定义：
- `include/db/official_data_repository.h`
- `src/db/official_data_repository.cpp`

### 2.2 当前运行时消费链

1. `PetFactory::initialize_runtime_data(...)`
   - 初始化只读 SQLite
   - 位置：`src/entities/pet_factory.cpp`

2. `PetFactory::create_pet(...)`
   - 读取 `MonsterRecord`（基础属性、种族值、可学技能）
   - 读取 `SoulMarkRecord`
   - 位置：`src/entities/pet_factory.cpp`

3. `Skills::loadSkills()`
   - 读取 `SkillRecord`
   - 将技能 side effect 拆分成 `SkillEffectRecord`
   - 位置：`src/entities/skills.cpp`

4. `SkillExecutionEffect::registerBranch(...)`
   - 按时点注册具体 effect（技能效果时点 / 攻击伤害时点）
   - 动态绑定施法方和目标方参数
   - 位置：`src/effects/continuousEffect.cpp`

5. `EffectFactory`
   - effect id -> `EffectFn` 映射表
   - 当前版本使用动态库注册（插件模式）
   - 位置：`include/effects/effect.h`, `src/effects/effect.cpp`

## 3. 数据表 -> 运行时字段映射

### 3.1 技能主链

- `moves`
  - `id/name/type_id/category/power/accuracy/priority/max_pp/cd`
  - `side_effect` + `side_effect_arg`：效果 ID 与参数串
- `side_effect`
  - `id/arg_count`：每个 effect 需要多少参数
- `effect_info`
  - `id/args_num/info/param/sp`：效果模板与参数语义

映射到结构：
- `SkillRecord`
- `SkillEffectRecord`（含 `effect_id/arg_count/args/effect_args_num/info/param/sp`）

### 3.2 精灵主链

- `monsters`
  - `id/name/type/type2/AddSeParam/Gender/HP/Atk/Def/SpAtk/SpDef/Spd`
  - `LearnableMoves.Move[]`
- `new_se`
  - `idx/stat/effect_id/args/can_reset/desc/intro/star_level`

映射到结构：
- `MonsterRecord`
- `LearnableMoveRecord`
- `SoulMarkRecord`

## 4. 新增“某只精灵效果”标准流程

以精灵 ID 为输入，建议按以下顺序：

1. 查精灵与技能池
   - SQL 或脚本：`scripts/tools/query_monster_profile.py`

2. 列出技能对应 effect id 与参数
   - 先看 `moves.side_effect / side_effect_arg`
   - 再补 `side_effect.arg_count` 与 `effect_info` 模板

3. 实现 effect 逻辑
   - 技能效果插件：`resources/moves_lib/lib_1.cpp`
   - 注册函数：`skill_plugin_register`

4. 确认 effect 注册时点
   - `src/entities/skills.cpp` 中 `default_register_state_for_effect(...)`
   - 与伤害强耦合的 effect 应落在 `BATTLE_*_ATTACK_DAMAGE`

5. 校对先后手与参数绑定
   - `src/effects/continuousEffect.cpp`
   - 使用 `state_for_owner(...)` 和 `bind_participants(...)`

6. 补 smoke 测试
   - `test/test.cpp`（建议按“注册时点 + 伤害快照变化 + 最终 hp”三层断言）

## 5. 常用查询命令（Agent 直接可用）

```bash
# 按精灵名输出技能/效果摘要
python3 scripts/tools/query_monster_profile.py --name "波克尔"

# 结构化输出，便于二次消费
python3 scripts/tools/query_monster_profile.py --name "波克尔" --json
```

```sql
-- 查技能主信息
SELECT id, name, side_effect, side_effect_arg
FROM moves
WHERE id = 10057;

-- 查 effect 模板
SELECT id, args_num, info, param, sp
FROM effect_info
WHERE id IN (4,5,6,7,8);

-- 查精灵可学技能
SELECT json_extract(json_each.value, '$.ID') AS move_id,
       COALESCE(json_extract(json_each.value, '$.LearningLv'), 0) AS learning_lv
FROM monsters, json_each(monsters.raw_json, '$.LearnableMoves.Move')
WHERE monsters.id = 12
ORDER BY learning_lv ASC, move_id ASC;
```

## 6. 交接时必须先检查的风险

1. 插件是否真的加载成功
- `EffectFactory::getInstance(lib_dir)` 首次调用必须带动态库目录。
- 仅调用 `getInstance()`（空目录）会导致 effect 缓存为空，技能效果无法注册。

2. macOS 动态库后缀
- 目前代码扫描后缀是 `.so`，在 macOS 通常应支持 `.dylib`。
- 如未处理，插件发现会失败。

3. smoke 二进制是否过期
- 先确认 `test/test` 的时间戳是否晚于源码；若过期，先重编译再看结果。

4. “留 1 血”类效果的落点
- 优先改 `resolvedDamage.final`，避免直接改 `target.hp` 后被后续统一扣血再次覆盖。

## 7. 推荐接手顺序（最小闭环）

1. 跑数据库更新脚本，确保 SQLite 与 raw 对齐
2. 用 `query_monster_profile.py` 抽出目标精灵 effect 清单
3. 在插件中实现缺失 effect，并注册
4. 在技能加载层确定 effect 时点
5. 在 FSM 跑一条 smoke（单技能、单时点）
6. 最后补先后手互换场景
