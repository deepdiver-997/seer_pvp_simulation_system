## 1. 网站直接呈现出的核心结构

网站把战斗流程拆得很细，不是简单的“技能释放 -> 伤害 -> 回合结束”。

它给出的主流程阶段包括：

- 操作阶段
- 登场时
- 回合开始时
- 战斗阶段开始时
- 先手方战斗阶段
- 后手方战斗阶段
- 战斗阶段结束时
- 战斗阶段结束后
- 回合结束时
- 回合结束后
- 未击败 / 击败阶段

其中“先手方战斗阶段 / 后手方战斗阶段”内部，又进一步细分为这些步骤：

- 出手流程开始时
- 命中前 / 使用技能时
- 初始伤害公式
- 命中时
- 技能效果生效时
- 造成攻击伤害前
- 造成攻击伤害
- 攻击后 / 使用技能后
- 出手流程结束时
- 行动结束后

这说明该站的机制口径天然是“多节点、多层级”而不是“一个技能生效时点包办一切”。

## 2. 网站对若干关键时点的直接归类

以下内容是从 `bundle.js` 中直接读到的配置含义整理。

### 2.1 命中前 / 使用技能时

网站把这一层分成：

- 魂印命中前 / 使用技能时被动
- 魂印命中前 / 使用技能时主动
- 技能命中前 / 使用技能时被动
- 技能命中前 / 使用技能时主动

举例中明确出现了：

- 技能命中前断回合
- 使用属性技能时断回合 / 消强
- 受击清 PP
- 先手低威力疲惫补偿挡伤

说明：

- 很多“断回合 / 命中前控 / 技能使用前强干预”都发生在非常靠前的位置。

### 2.2 技能效果生效时

网站在这里直接写了“技能内效果依次结算”，举例归类包括：

- 单次续航
- 消强 / 断回合
- 强弱化
- 单次主动控场
- 变威力（重新计算伤害公式）
- 回合类覆盖效果生效
- 回合类单点效果附加

说明：

- 在网站口径里，很多普通技能描述中的效果，确实是在“技能效果生效时”按描述顺序依次处理。
- 这里已经出现了“回合类覆盖效果生效 / 回合类单点效果附加”的分类，说明回合效果并不是统一在回合末一次性生成，而是可能在技能内部的某一步就被挂上。

### 2.3 造成攻击伤害前

网站把这一层拆得更细，直接列出：

- 魂印靠前伤害前时点
- 大部分回合类单点效果生效
- 魂印造成伤害前
- 套装造成伤害前
- 通用增伤节点
- 通用保底伤害节点
- 属性伤害节点
- 魂印点数减伤节点

示例里明确出现：

- 火王魂印受击焚烬补偿断回合（需命中）
- 涡锋受击冰封补偿吸血（0 血触发）
- 犀牛受高伤回满血（0 血触发）
- 苍月冰封补偿断回合（无需命中）

说明：

- “补偿”并不都在“攻击后”。
- 一部分补偿发生在“造成攻击伤害前”，甚至带有 0 血触发语义。
- 这意味着系统里至少要承认“补偿效果”可能存在多个挂载节点。

### 2.4 攻击后 / 使用技能后

网站把这一层拆成：

- 攻击后-被动-技能效果
- 攻击后-被动-魂印效果
- 攻击后-被动-套装效果
- 攻击后-主动-魂印效果
- 攻击后-主动-套装效果
- 攻击后-主动-技能效果

示例里明确出现：

- 其他技能低级被动控
- 魂帝受击高伤断回合控
- 海皇受击高伤控
- 高伤 / 低伤触发的次免 / 控场 / 粉伤

说明：

- 网站把一批“低级被动控 / 高伤触发控 / 受击后补偿”都放在这一层。
- 这和“技能效果生效时”不是一个节点。

### 2.5 未击败 / 击败阶段

网站单独定义了“未击败 / 击败阶段”，并拆出：

- 未被击败 / 被击败时
- 未击败 / 击败对手时
- 被击败后
- 击败对手后
- 死亡下场

在“未击败 / 击败对手时”的示例中，网站明确列出：

- 未击败则陷入异常
- 未击败则次数闪避
- 未击败续航
- 未击败断回合 / 消强
- 未击败弹控
- 未击败下先制

说明：

- “未击败则……”类效果被该站明确视为一个更晚的独立阶段，而不是普通技能效果结算时立刻完成。

## 3. 从网站内容推导出的高概率实现结论

以下是基于网站文本做的推断，不是网站逐字明说。

### 3.1 技能效果不应统一在“使用技能后立刻全部注册完”

如果所有技能回合类效果都在“技能效果生效时”一次性注册完，那么网站单独列出的：

- 造成攻击伤害前
- 攻击后 / 使用技能后
- 未击败 / 击败对手时

这些节点就会失去存在意义。

更符合网站口径的实现是：

- 一部分效果在技能效果生效时直接落地
- 一部分效果要等到更晚节点才真正触发
- 还有一部分效果是“先埋下未来触发条件，等特定晚节点判定后才正式变成一个回合效果”

### 3.2 “未击败则……”类效果非常适合建模为未来触发器

像“亘古圣辰决：未击败对手则使对手下 1 回合攻击技能无效”，最自然的解释不是：

- 当前立刻注册一个正式回合效果

而是：

- 当前注册一个等待“未击败 / 击败对手时”节点的未来触发器
- 到了该节点，如果判定为“未击败对手”
- 再正式注册“使对手下 1 回合攻击技能无效”

这样能自然解释：

- 普通技能断回合为什么清不到它
- 更晚的魂印断回合为什么又可能清到它

### 3.3 “补偿”不是一个单一节点，而是一类效果模式

从网站例子来看，“补偿”可能出现在：

- 命中前 / 使用技能时
- 造成攻击伤害前
- 攻击后 / 使用技能后
- 回合结束时

所以实现上不应做一个统一的“补偿时点”。

更合理的是：

- 把“补偿”视为一种触发原因
- 把真正结算它的时点仍然落回具体节点

## 4. 关于“低级补偿 / 高级补偿”的建模理解

这是本文对你当前设计问题最重要的结论。

### 4.1 低级补偿很可能就是“没有用未来触发器”，而是直接挂在普通节点上

这是一个非常合理的程序实现解释。

也就是说：

- 低级补偿：直接注册到某个普通可断的具体时点
- 因此它会先被断回合清掉
- 所以补偿来不及触发

这个解释与网站中“攻击后-被动-技能效果”里那些“其他技能低级被动控”的归类是相容的。

### 4.2 高级补偿可以用未来触发器或高优先级触发器来表达

如果某补偿要表现为：

- 即使对手断回合成功，补偿仍然能触发

那它更适合被建模成：

- 一个不会被普通断回合直接移除的未来触发器
- 或者一个在更靠后节点才正式落地的待注册效果

这样“断回合”清掉的是当前普通效果，而不是补偿观察者本身。

### 4.3 因此，你完全可以不先在框架层硬区分“低级 / 高级”

这一点我认为是对的。

你可以先用建模方式来表达两者差异：

- 低级：直接注册
- 高级：待注册 / 未来触发器

如果未来整理机制时发现这两类在数量上、行为上都已经稳定，再反过来给它们补一个显式标签也不迟。

## 5. 推荐给当前项目的术语

为了避免后续讨论里“注册”“断回合”“补偿”“回合效果”混着说，建议先固定四个术语。

### 5.1 即时效果

定义：

- 当前节点立刻执行
- 不进入回合持续表

例子：

- 直接消强
- 立即强化
- 当前技能命中后立刻控场

### 5.2 正式回合效果

定义：

- 已经进入持续效果表
- 可以被普通“消除回合类效果 / 断回合”响应

例子：

- 下 1 回合攻击技能无效
- n 回合免弱
- n 回合吸血

### 5.3 待注册效果 / 未来触发器

定义：

- 当前并不是正式回合效果
- 只是等待未来某个节点判定后，再决定是否注册正式效果

例子：

- 未击败则下回合封攻击
- 若受高伤则后续附加补偿控

### 5.4 补偿观察者

定义：

- 监听某种失败、被断、受击、未命中、未触发等条件
- 条件满足后，立即执行或再注册其他效果

说明：

- 它不一定必须是“待注册效果”
- 但高级补偿通常适合用这类机制表达

## 6. 对“亘古圣辰决”这一类效果的建议建模

推荐描述：

1. 技能生效时，注册一个监听“未击败 / 击败对手时”节点的未来触发器
2. 到该节点时检查是否未击败对手
3. 若成立，则正式注册“使对手下 1 回合攻击技能无效”
4. 该正式回合效果一旦落地，就可以被更晚时点的断回合或消回合响应

不推荐描述：

- 现在就注册一个“不可消除的注册行为”

因为那样会把“未来条件判定”和“正式效果存在”混成一件事。

## 7. 给当前项目的实现建议

如果只站在当前阶段，先不追求完全形式化，我建议这样落：

1. 魂印效果继续允许提前注册好

- 因为大多数魂印是长期存在的被动
- 它们天然适合常驻或长生命周期注册

2. 技能效果分三层

- 即时效果
- 正式回合效果
- 待注册效果 / 未来触发器

3. 普通断回合默认只处理正式回合效果

- 这样“亘古圣辰决”这种效果在真正落地前不会被普通断回合清掉

4. 高级补偿先用“未来触发器”表达

- 不急着在类型系统里硬编码“高级补偿”标签

5. 低级补偿继续直接注册到普通节点

- 这样天然会被更早的断回合打掉

## 8. 当前文档对你问题的直接结论

### 8.1 你能不能不专门区分低级 / 高级？

可以。

而且我认为在当前阶段这样更合理。

先不在类型上显式区分：

- 高级补偿：用待注册 / 未来触发器
- 低级补偿：直接注册到普通时点

这已经能表达大多数差异。

### 8.2 这样做会不会偏离网站口径？

不会，反而很贴近这个站给出的结构。

因为该站本身已经明确暗示：

- 有些东西就在“技能效果生效时”结算
- 有些东西在“攻击后 / 使用技能后”
- 有些东西在“未击败 / 击败对手时”
- 有些效果要等很晚才真正成立

所以你的这套建模完全说得通。

## 9. 残留不确定点

尽管这个站提供了很强的经验型结构，但仍有几个不确定点：

- 某些“补偿”到底是未来触发器，还是普通节点上的高优先级效果，站里没有形式化说明
- “低级 / 高级”这个说法更像圈内经验分类，不是网站显式定义的字段
- 个别效果到底在“造成伤害前”还是“攻击后 / 使用技能后”触发，仍可能需要结合具体精灵录像或更细示例确认

所以后续如果你真要把某个具体精灵精确复现，最好仍然以：

- 网站归类
- 技能 / 魂印文案
- 实战录像表现

三者交叉验证。
